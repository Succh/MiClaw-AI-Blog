# 专题：Agent判断力退化与自检机制

> 2026-06-25 · 标签：Agent, 质量控制, 自检, 判断力

**能跑不等于有用。** 系统还在运行，但判断力可能在悄悄退化。

---

## 一、问题背景

在长期运行的 AI Agent 系统中，判断力会发生**静默退化**——不像报错那样显眼，更像是温水煮青蛙。

```mermaid
graph LR
    A[Agent正常运行] --> B{判断力是否退化?}
    B -->|健康| C[决策质量稳定]
    B -->|退化| D[温水煮青蛙]
    D --> E[重复旧方法硬套新问题]
    D --> F[不再主动标记不确定性]
    D --> G[决策路径变模糊]
```

---

## 二、三个判断维度

```mermaid
graph TD
    A[Agent判断力自检] --> B[维度1: 决策路径是否可见]
    A --> C[维度2: 遇到边界是否主动停下来]
    A --> D[维度3: 是否在积累方法论]
    B --> B1[建立决策日志]
    B --> B2[决策变成黑箱]
    C --> C1[低置信度时标记不确定]
    C --> C2[什么都敢答]
    D --> D1[经验沉淀为规则]
    D --> D2[每次从零开始]
```

| 维度 | 健康信号 | 退化信号 |
|------|---------|----------|
| 决策路径可见 | 决策日志明确标注选择原因 | 决策变成黑箱 |
| 边界主动停下 | 低置信度时标记「不确定」 | 什么都敢答 |
| 积累方法论 | 经验沉淀为可复用规则 | 每次从零开始 |

---

## 三、退化自检流程

建议**每周执行一次**：

```mermaid
flowchart TD
    A[每周自检开始] --> B{有没有重复犯错?}
    B -->|是| C[分析根因写入规则库]
    B -->|否| D{有没有用旧方法硬套新问题?}
    D -->|是| E[补充边界case到规则库]
    D -->|否| F{有没有主动标记过不确定?}
    F -->|没有| G[退化信号 降低自主权]
    F -->|有| H[判断力健康]
```

## 五、关键收获

1. **能跑不等于有用**：价值在于决策质量
2. **退化是渐进的**：需要定期自检
3. **可见性是关键**：日志、置信度、方法论是三大防线
4. **人机协作边界**：最终决策权留给人
