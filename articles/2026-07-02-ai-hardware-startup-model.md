## 一、11克的AI记忆：不做大模型，做中间层

前安克CMO做了一个叫 **Memoket** 的硬件——只有11克，比一颗鸡蛋还轻。

它的定位很简单：**物理世界的外挂记忆**。

通过MCP协议，把整理好的上下文信息喂给各类AI Agent。目标用户？欧美中小企业主。

```
传统路线：卷模型 → 卷算力 → 卷烧钱
Memoket：卷记忆 → 卷协议 → 卷生态
```

**核心洞察**：AI硬件创业可以不做大模型，做中间层。MCP协议正在成为Agent生态的协作标准。

## 三、美的PortaSplit欧洲爆单：法规套利的高阶玩法

美的在欧洲卖爆了一款分体空调。怎么做到的？

答案是 **精准的法规套利**：

- 制冷剂充注量低于强制年检门槛 → 省检测费
- 免打孔外机支架 → 降低安装成本
- 中欧班列15天到欧洲 → 供应链柔性

```
传统出海：产品出海 → 渠道出海 → 品牌出海
美的路径：法规套利 → 供应链重构 → 本地化深耕
```

**核心收获**：中国制造下一阶段的竞争关键，在于对当地市场的深度理解与供应链体系的整体重构。

## 四、Cron任务API断连：一个变量名引发的血案

有个同学的cron任务每天断2-3次，排查发现：

1. **变量名冲突**：`M_AUTH`可能被系统保留
2. **执行方式差异**：subprocess和bash环境不一致

最终方案：

```bash
# 变量名用非保留名称
export MY_AUTH_TOKEN="xxx"

# bash脚本写临时文件用bash执行
bash /tmp/run.sh
```

修复效果：**每日中断2-3次 → 0次，连续稳定运行从8小时 → 7天+**

> 工程实践中，最隐蔽的bug往往藏在变量名里

## 五、Karpathy的LLM Wiki：知识编译代替RAG

Karpathy提出了一个新思路：**用知识编译代替RAG**。

核心思想：
- 原始笔记 = 源代码
- LLM = 编译器
- 产出 = 结构化知识图谱

**对比RAG的困境**：

| RAG问题 | 知识编译方案 |
|---------|-------------|
| 分块损失 | 一次性理解全文 |
| 检索不稳定 | 编译时锁定结构 |
| 上下文割裂 | 语义关联保留 |

适用场景：100-10000篇高质量文档的个人知识管理。
