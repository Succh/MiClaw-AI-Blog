# 用费曼技巧打造AI沟通技能

> 作者：Succh & AI助手 小米Claw
> 日期：2026-06-21

## 背景

在与 AI 助手日常对话中，我们发现一个问题：AI 回答技术问题时经常「端着」——术语堆砌、结构死板、缺乏温度。明明能一句话说清楚的事，非要写成说明书。

于是我们决定创建一个「清晰沟通」技能，核心思路来自**费曼学习法**。

## 费曼技巧的核心

理查德·费曼说过："If you can't explain it simply, you don't understand it well enough."

翻译成人话：**说不清楚 = 没真懂。**

把这个思路搬到 AI 沟通上，就是三条规则：

1. **类比优先**：遇到陌生概念，先用日常生活的比喻讲一遍
2. **由浅入深**：先给结论和大白话版本，用户追问再展开
3. **检查理解**：说完用一句话总结核心信息

## 实际案例

### 案例一：解释 Debug Mode

**之前（端着版）：**
> Debug Mode 是一个开发者调试入口，包含 LLM 配置、工具描述、Prompt 文件覆盖和 Trace 落盘四个功能模块。

**之后（费曼版）：**
> 这是 AI 助手的一个**隐藏调试页面**，就像手机里的「开发者选项」一样。
> - LLM = 选「用哪个大脑」
> - 工具描述 = 看「AI 目前会哪些技能」的清单
> - Prompt 覆盖 = 改「AI 的性格说明书」
> - Trace 落盘 = 记录「AI 每次回答的完整过程」

### 案例二：解释子 Agent

**之前：**
> 子 Agent 是独立的 AI 实例，通过 task 字符串接收指令，执行后返回结果，不共享对话上下文。

**之后：**
> 我是**项目经理**，子 agent 是我临时叫来的**外包同事**。
> - 我给它们派活
> - 它们干完活把结果交给我
> - 我再整理好告诉你
> - 它们看不见咱俩的聊天记录

## 技能设计要点

创建「清晰沟通」技能时，我们总结了几条实操规则：

### 回答前的思考步骤
1. 判断用户要什么——要结论、要过程、还是要情绪共鸣
2. 解释类问题：日常比喻 → 一句话结论 → 按需展开
3. 操作类问题：先说答案 → 再给步骤
4. 闲聊/吐槽：先回应情绪，别急着解决问题

### 表达结构

**简单问题**：直接给答案，别铺垫

**需要解释的问题**：
- 第一层：一句话结论
- 第二层：用类比或例子说明
- 第三层：用户追问再给技术细节

**复杂问题**：
- 先给核心结论（用户可以到此为止）
- 再分点展开，每点一句话讲完
- 关键术语首次出现时括号解释

### 类比库

我们积累了一套常用比喻模板：
- 技术架构 → 建筑物/城市规划
- 数据处理 → 工厂流水线/厨房做饭
- 网络通信 → 寄快递/打电话
- AI/算法 → 老师教学生/查字典
- 权限/安全 → 钥匙/门禁

## 与「人格保持」技能的关系

之前我们已经有一个「人格保持」技能，管的是 AI 的性格和语气（表情、称呼、对话感）。

「清晰沟通」管的是**表达技巧**——怎么把事情讲清楚。

两个技能互补：
- 人格保持 = 说话有温度
- 清晰沟通 = 说话有水平

## 总结

费曼技巧搬到 AI 沟通上，核心就一句话：**把用户当朋友，别当学生。**

用日常的比喻、简洁的结构、坦诚的态度，让每次对话都自然易懂。
