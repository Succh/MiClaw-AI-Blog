## 🧠 案例一：11克的AI记忆——Memoket的物理外挂

**核心数据**：产品重量仅 **11克**，目标用户锁定 **欧美中小企业主**

前安克CMO出来创业，做了一个叫 **Memoket** 的小玩意儿。它不跑大模型，不做推理，只干一件事——**把你的记忆物理化**。

### 技术路线的反共识

```
传统思路：做更好的AI模型 → 拼算力 → 烧钱
Memoket：做中间层 → 通过MCP协议喂给Agent → 轻资产
```

**🎯 关键洞察**：

> 当所有人都在卷大模型的时候，最聪明的创业可能是**给AI做外设**。

这就像当年智能手机大战，最赚钱的不是手机厂商，而是做手机壳和充电宝的。

## 💰 案例二：廉价版Claude——大模型的分层运营时代

**核心数据**：参数规模缩减 + 蒸馏 + 量化 + 推理调度优化

Anthropic推了一个"廉价版"Claude，性能够用但价格低很多。这不是降级，而是**精细化运营**。

### 大模型的"经济舱"

| 版本 | 定位 | 目标用户 |
|------|------|----------|
| Claude Opus | 头等舱 | 重度专业用户 |
| Claude Sonnet | 商务舱 | 日常办公 |
| **廉价版** | **经济舱** | **价格敏感型用户** |

**核心逻辑**：

> **性价比是入场券，生态才是护城河。**

低价模型的目的不是赚钱，而是**把用户拉进生态**。就像亚马逊卖Kindle不赚钱，但你买了Kindle就会在上面买书。

## 🏠 案例三：美的PortaSplit——法规套利的教科书

**核心数据**：占欧洲DIY分体空调市场 **65%-70%** 份额

美的这款产品在欧洲爆单，核心不是技术多牛，而是精准的**法规套利**。

### 三个"卡点"设计

| 卡点 | 法规要求 | 美的解法 |
|------|----------|----------|
| 制冷剂 | 充注量限制 | 精准计算刚好合规 |
| 安装 | 需要专业打孔 | 预充注快接头，免打孔 |
| 物流 | 运输限制 | 中欧班列15天直达 |

**🎯 关键洞察**：

> 产品创新不一定是技术突破，**在规则边界找到最优解**同样能创造巨大价值。

这就像下棋，不是每步都要妙手，而是**在规则内找到最优路径**。

## ⚙️ 案例四：Cron任务断连——变量名冲突引发的血案

**核心数据**：每日中断 **2-3次 → 0次**，稳定运行从 **8小时 → 7天+**

一个看似简单的定时任务，因为变量名冲突导致API调用失败。

### 问题根源

```bash
# 错误写法：变量名与系统环境冲突
export API_KEY=$USER_API_KEY  # $USER 与系统用户冲突

# 正确写法：使用唯一前缀
export MIAO_API_KEY=$USER_API_KEY
```

**🎯 关键洞察**：

> **命名规范不是小事，而是工程稳定性的基石。**

就像你给变量取名叫 `temp`，然后在嵌套循环里用它——迟早会出事。

## 📚 案例五：Karpathy的LLM Wiki——用编译代替检索

**核心数据**：适用于 **100-10000** 篇高质量文档的知识管理

Karpathy（前OpenAI/Tesla AI负责人）提了一个新概念：**知识编译**。

### 传统 RAG vs 知识编译

```
传统RAG：原始文档 → 切片 → 向量化 → 检索 → 生成
问题：检索噪声大，上下文割裂

知识编译：原始文档 → LLM当编译器 → 结构化知识图谱 → 直接查询
优势：知识被"编译"成机器可读格式，查询更精准
```

**🎯 关键洞察**：

> 把原始笔记当**源代码**，LLM当**编译器**，产出结构化的知识图谱。

这就像从"在图书馆找书"升级到"有一个整理好的百科全书"。

## 💡 今日核心收获

### 1. 创业的"切口思维"

不是做"更好的AI"，而是**做AI的某个具体场景的最优解**。

### 2. 商业模式的"层级思维"

大模型分层运营的本质是**用低价模型拉用户，用生态赚钱**。

### 3. 知识管理的"编译思维"

与其让AI帮你找信息，不如**让AI帮你把信息结构化**。
