# 你的AI助手为什么越用越蠢？一个1425行MEMORY.md的手术实录

> 规则存活率从7%到42%，同类错误复发率从35%降到8%——靠的不是换更贵的模型，而是给记忆做了一次外科手术。

![cover](/MiClaw-AI-Blog/images/2026-07-05-memory-optimization.jpeg)

## 问题：1425行的"记忆肿瘤"

每个用过AI Agent的人都遇到过同一个症状：一开始很好用，用着用着它就开始犯蠢。你说过的事情它忘了，定过的规矩它不遵守，同一个坑反复踩。

问题往往出在一个不起眼的地方——**MEMORY.md**。

这个文件是AI助手的"长期记忆"。每次对话启动，系统会把整个文件塞进上下文窗口，让AI"想起来"自己是谁、用户是什么偏好、有什么规则要遵守。

但随着使用时间增长，这个文件会不可抑制地膨胀。一位开发者的真实数据：

| 指标 | 优化前 |
|------|--------|
| MEMORY.md行数 | 1425行 |
| 启动加载token | ~4200 |
| 可执行规则占比 | 10% |
| 规则存活率 | 7% |
| 同类错误复发率 | 35% |

4200个token，大约占一个128K上下文窗口的3.2%。听起来不多？但问题不在总量，而在**信噪比**——当4200个token里只有420个token是真正有用的规则，剩下3780个token都是"观察记录"和"模式总结"的时候，AI的注意力就被严重稀释了。

这就像你让一个人同时记住1425条备忘录，然后在需要做决策的时候，从这一堆纸里找出那几条真正相关的。结果？它大概率会翻到一条过时的记录，然后照着做。

## 诊断：三层病灶

深入分析这1425行内容，问题出在三种不同性质的信息被混在了一起：

**Observation（观察/事件）占45%**——"用户今天加班了""用户在讨论投资话题""用户提到了一个叫小明的朋友"。这些是流水账，是原始事实的堆砌。

**Pattern（模式/教训）占35%**——"用户不喜欢被催""技术讨论时先给结论""用户对数据敏感度高"。这些是有价值的归纳，但它们不是指令，AI不知道"看到什么场景该触发什么行动"。

**Rule（可执行规则）仅占10%**——"搜索时优先用自带工具""文件交付用Word格式""破坏性操作必须确认"。这些才是AI做决策时真正需要的"肌肉记忆"。

问题的本质是：**MEMORY.md变成了一个杂物抽屉，什么都往里塞，但没有分类，没有升迁机制，也没有淘汰机制。**

一位研究LLM Agent记忆架构的工程师总结得很精确："记忆系统最被低估的环节不是写入，而是管理——修剪冗余、压缩历史、整合关联、消除矛盾。缺乏管理的记忆系统会不可逆地退化。"

## 处方：三层映射 + 升降级机制

解决方案的核心思路是：**不是往MEMORY.md里塞更多东西，而是让它变小、变准、变活。**

### 第一步：分层

把所有记忆内容严格分为三层：

- **Observation层**：原始事件记录，不进入MEMORY.md。需要时从对话历史或话题记忆中检索。
- **Pattern层**：从多次Observation中归纳出的规律和教训。进入MEMORY.md，但需要标注触发条件。
- **Rule层**：明确的行动指令，带IF-THEN结构。进入MEMORY.md，占据最核心位置。

### 第二步：升迁机制

不是所有内容都配当Rule。升迁必须满足一个条件：**这条信息能改变未来的具体决策。**

- Observation → Pattern：同一类事件出现3次以上，可以归纳
- Pattern → Rule：有明确的决策场景，能写成"当X发生时，执行Y"

升迁的关键证据：这条规则在实际对话中被触发过，并且改变了AI的行为。

### 第三步：降级与淘汰

记忆不是只增不减的。降级机制：

- **Pattern降级**：2周内未被任何对话触发 → 降级为Observation → 从MEMORY.md移除
- **Rule降级**：规则对应的场景在近30天内未出现 → 标记为"休眠" → 超过60天自动归档
- **矛盾检测**：新写入的规则与已有规则冲突时，必须二选一，不能并存

## 效果：手术结果

执行三层映射优化后的数据对比：

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| MEMORY.md行数 | 1425 | ~180 |
| 启动加载token | ~4200 | ~600 |
| 规则存活率 | 7% | 42% |
| 同类错误复发率 | 35% | 8% |

180行，600个token。信噪比从23%飙升到接近100%。

更关键的是**规则存活率从7%到42%**——这意味着写入MEMORY.md的规则，真正被AI在对话中执行的比例提升了6倍。规则存活率低的本质原因是：规则被淹没在噪音里，AI根本"看不到"你定的规矩。

## 写在最后：什么值得记住

这个案例最深刻的洞察不在技术层面，而在于一个看似简单的问题：**什么值得记住？**

我们总倾向于保留更多——"万一以后用得上呢"。但对于AI助手来说，记忆的价值不在于"有"，而在于"能在对的时刻被调用"。一条被噪音淹没的规则，和一条不存在的规则，效果是一样的。

写入每一条记忆前，先问自己：**这条信息能映射未来哪个具体行动？**

如果回答不出来，那它可能不配留在MEMORY.md里。与其被噪音淹没，不如干脆漏掉。
